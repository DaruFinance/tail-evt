# 2026-04-26 — A sign flip in the GPD probability-weighted-moments estimator

When I first built the synthetic-mode end-to-end test for `evt-cli`, every shape
estimate came back at the clamp boundary — `ξ̂ = -0.9` regardless of the input.
That is a textbook symptom of a mis-derived PWM formula, not a numerical issue,
so the fix is interesting enough to be worth writing down.

## What I had

The original (incorrect) closed form in the code, with `b₀ = (1/n) Σ x_(i)` and
`b₁ = (1/n) Σ (i/(n−1)) · x_(i)` (0-based ascending sort, i = 0…n−1):

```rust
let xi = 2.0 - b0 / (b0 - 2.0 * b1);
let beta = b0 * (1.0 - xi);
```

This is the Hosking–Wallis 1987 expression *with the sign of the second term
flipped*. With the wrong sign, `xi` runs in the opposite direction and pegs at
the −0.9 clamp for any non-degenerate sample.

## The actual derivation

Two PWMs are needed:

```
α₀  = E[X]              → β / (1 − ξ)
α_F = E[X · F(X)]       → β / (1 − ξ)  −  β / (2(2 − ξ))
```

(The second identity is Pickands 1975 algebra; integrate by parts.)

Inverting:

```
ξ = 2 + α₀ / (α₀ − 2 α_F)
β = α₀ · (1 − ξ)
```

The `+` is the part the code had backwards. The HW87 form quoted in most
textbooks is for the *(1 − F̂)*-weighted PWM (`b₁ = (1/n) Σ ((n−1−i)/(n−1)) · x_(i)`);
the ascending-sort code variant uses `F̂` weights, which flips the sign on the
second term.

A quick exponential sanity check (ξ = 0):

```
α₀ = β
α_F = β − β/4 = 3β/4
α₀ − 2 α_F = β − 3β/2 = −β/2
α₀ / (α₀ − 2 α_F) = β / (−β/2) = −2
ξ = 2 + (−2) = 0   ✓
```

## How I caught it

The end-to-end smoke test (`tests/smoke.rs`) was added before this fix landed,
so it ran against the old formula and reported finite-but-wrong shapes. I only
realised something was off when I cross-checked one synthetic asset against
SciPy's `genpareto.fit` MLE in Python — the MLE returned ξ ≈ +0.15 against the
PWM's −0.9, which does not happen by accident. From there a four-line
derivation showed which sign was wrong.

## Reference

- Hosking, J. R. M. & Wallis, J. R. (1987). *Parameter and quantile estimation
  for the generalized Pareto distribution.* Technometrics 29(3), 339–349.
- Pickands, J. (1975). *Statistical inference using extreme order statistics.*
  Annals of Statistics 3(1), 119–131.
